## Summary

**Goal:** Apply the whole-repo maintenance review by reorganizing high-churn internals, reducing Go-idiom smells, and improving maintainability without breaking the v2 public import surface.

**Architecture:** Keep existing v2 exported function signatures and import paths compatible, then improve organization behind those APIs. Split the large provider contract into focused service interfaces while preserving `core.Provider`, split the paper backend implementation into files by responsibility, extract layout/pagination ownership from the root `Paper` methods, extract rich-text layout into pure helpers, add normalized-copy paths for props/config, add a programmatic render-issue report path, and document boundaries around third-party-derived PDF backend code. Breaking API cleanup is recorded as deferred v3 work.

**Tech Stack:** Go 1.26.1, existing standard-library-only refactors, existing mockery-generated mocks, existing `golangci-lint`, `go vet`, `go test`, rendered-output regression checks, and race tests for high-risk shared-state paths.

## Scope

### In Scope

- Split `pkg/core.Provider` into smaller focused interfaces embedded by the existing `Provider` interface.
- Move `internal/providers/paper/provider.go` methods into focused files by responsibility: document lifecycle, metadata, text, images, codes, links/shapes/capabilities, and provider construction.
- Extract root package layout/pagination state into a private page-builder type so `Paper` is less of a layout god object while preserving behavior.
- Refactor `internal/providers/paper/richtext.go` so tokenization, measurement/layout, and rendering are separate testable units.
- Add normalized-copy helpers in `pkg/props` and use them from components/config so defaulting is explicit and less order-dependent.
- Keep existing exported constructor signatures in v2; add opt-in concrete-returning alternatives only where they do not replace existing functions.
- Add value-returning color helpers and switch internal defaults away from pointers to exported mutable global color variables.
- Add a programmatic render issue/report path for provider operations that currently draw fallback error text.
- Reorganize `pkg/html/css.ComputedStyle.ApplyCtx` around grouped handler helpers while keeping the exported flat `ComputedStyle` struct and keyed literals compatible.
- Add public package documentation for aggregate packages instead of removing public empty import paths in v2.
- Add ownership documentation for `internal/paperpdf` so it is treated as third-party-derived backend code, not normal project logic.
- Add guard tests for the new organization: rich-text layout behavior, constructor compatibility, provider subinterface coverage, normalized props behavior, render issues, and rendered-output regression coverage.

### Out of Scope

- Removing existing v2 import paths such as `pkg/components/text`, `pkg/consts/align`, `pkg`, or `pkg/core`. That is a v3 API cleanup because users may import them directly.
- Changing existing exported constructor return signatures such as `paper.New`, `config.NewBuilder`, or component `New` functions. Exact signature changes are a v3 cleanup.
- Moving exported fields out of `pkg/html/css.ComputedStyle`; keyed literals such as `css.ComputedStyle{FontSize: 10}` must keep compiling in v2.
- Replacing mockery or regenerating all mocks as a goal. Existing mocks may be touched only when signature changes require it.
- Rewriting `internal/paperpdf` internals. That code is backend source internalized from GoFPDF and should stay isolated.
- Changing PDF rendering semantics, HTML/CSS support, fixture outputs, generated bytes, or documented v2 examples.
- Replacing retained domain dependencies.

## Context for Implementer

- **Patterns to follow:** The repo already uses compatibility-preserving narrow capability interfaces such as `core.RichTextProvider` in `pkg/core/richtext_provider.go` and compile-time provider assertions in `internal/providers/paper/provider.go`.
- **Conventions:** Keep public behavior stable. Prefer adding narrower internals over removing public packages. Use `apply_patch` for manual edits and `gofmt`/`goimports` for formatting.
- **Key files:** `pkg/core/provider.go` defines the large provider contract; `internal/providers/paper/provider.go` delegates backend features; `paper_layout.go` owns pagination; `internal/providers/paper/richtext.go` is the largest renderer hotspot; `pkg/html/css/computed.go` is the CSS property dispatcher; `pkg/components/*` constructors return interface types today; `pkg/props/*` contains mutable property defaulting.
- **Gotchas:** Mockery generated 23 files under `mocks/`; broad interface signature changes can force expensive regeneration. `internal/paperpdf` is excluded from lint/format in `.golangci.yml` and has its own license/notice. `props.BlackColor` and similar globals are exported mutable variables, so internal code should prefer value helpers but v2 should not remove the vars.
- **Domain context:** Paper builds a logical page tree first, then renders pages to a PDF backend. Concurrent generation renders page groups independently, so layout state and renderer state must not leak across providers or goroutines.

## Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
